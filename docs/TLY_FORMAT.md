# TLY 动态歌词格式

`.tly` 是 tnuxmusic 的可扩展歌词格式，不追求兼容 LRC，而是优先服务：

- 时间轴；
- 自动滚动；
- 行级翻译；
- 任意标签；
- 长句开始 / 结束时间。

## 基本结构

```tly
@title = Song
@artist = Artist
@lang = zh-CN

[00:12.340]主歌词
[00:12.340|tr=en]English translation
[00:18.000 --> 00:22.500|verse=1|mood=calm]带结束时间和标签的歌词
```

## 时间

支持：

- `mm:ss`
- `mm:ss.xxx`
- `hh:mm:ss.xxx`
- 小数点也可写成逗号：`00:12,340`

## 主歌词

```tly
[00:10.000]一句歌词
[00:10.000 --> 00:13.500]一句有结束时间的歌词
```

如果没有结束时间，播放器会用下一句歌词开始时间作为结束点。

## 翻译

翻译行使用同一个开始时间，并加 `tr` 属性：

```tly
[00:10.000]你好，世界
[00:10.000|tr=en]Hello, world
[00:10.000|tr=ja]こんにちは、世界
```

播放器会把同一时间点的翻译挂到主歌词下面。

## 标签

`|key=value` 或 `|tag` 都会被保留：

```tly
[01:02.000|verse=2|speaker=vocal|highlight]副歌开始
```

这些标签可以用于以后做角色歌词、逐字高亮、舞台提示、卡拉 OK 片段等扩展。

## 逐字 / 卡拉 OK 高亮

主歌词行里可以嵌入 `<时间>` 标记。播放器会把两个标记之间的文字当成一个高亮片段。

绝对时间：

```tly
[00:12.000]<00:12.000>逐<00:12.250>字<00:12.520>高亮
```

相对时间：以当前歌词行的开始时间为基准。

```tly
[00:16.000]<+00:00.000>相<+00:00.180>对<+00:00.360>时间
```

这一行的纯文本仍会被解析为：

```text
逐字高亮
相对时间
```

如果只写行级时间，不写行内 `<时间>`，播放器仍按整行滚动。
